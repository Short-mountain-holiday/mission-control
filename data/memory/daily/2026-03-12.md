## Security Review + Spec v3 (05:00 - 05:15 MYT / 3:00 - 3:15 PM CDT)

### What Happened
Micah asked about security for Mission Control. Did a full security audit of the spec.

### Critical Findings (Fixed in Spec v3)
1. **Shell injection** in `lib/openclaw.ts` — user input passed directly to `exec()`. Added `sanitizeShellArg()` + `sanitizePath()` functions
2. **Chat widget = full gateway access** — proxied to gateway with operator-level permissions
3. **No brute-force protection** on login — added rate limiting (5 attempts/15min/IP)
4. **No CSRF protection** — added origin verification on all mutation endpoints
5. **Cookie too long-lived** — reduced from 30 days to 7, added COOKIE_SECRET salt
6. **Error handling leaked internals** — standardized to generic client errors
7. **Path traversal in Docs** — added multi-layer validation

- **ContextEngine plugin interface** — new lifecycle hooks for custom memory strategies
- **lossless-claw plugin** — reference implementation that stores full conversation history without summarization
- Pluggable architecture — swap out default compaction logic with alternatives

### How It Works
- Lifecycle: bootstrap → ingest → assemble → compact → afterTurn → prepareSubagentSpawn → onSubagentEnded
- Default "LegacyContextEngine" preserves existing compaction
- Plugins like `lossless-claw` implement alternative strategies (full retention, data warehouse export, etc.)
- Installed via plugin system, configured in `openclaw.json`

### Status

## Mission Control Phase 2 Build Session (05:49 - 07:50 MYT / 3:49 - 5:50 PM CDT)

### Gateway Discovery: Sandbox Tools Can't Be Proxied
**Critical finding:** `read`, `write`, `edit`, `exec`, `process` are sandbox-only runtime tools. They do NOT exist in `createOpenClawTools()` and CANNOT be exposed via gateway HTTP `/tools/invoke` regardless of config. The Phase 2 spec was built on a wrong assumption.

**Available via gateway:** `memory_get`, `memory_search`, `web_search`, `web_fetch`, `message`, `browser`, `nodes`, `canvas`, `tts`, `session_status`, etc.
**NOT available:** `read`, `write`, `edit`, `exec`, `process`, `image`

### Fixes Applied
1. **Memory screen** → Rewrote to use `memory_get` (reads MEMORY.md + memory/*.md)
